为什么连外包公司都对你爱答不理？

发表于 2018-05-28   |   分类于 程序人生   |   阅读次数

随着互联网+口号和政策的推动，很多企业，尤其是传统企业都在转型，也越来越认识到互联网的重要性，都在想办法如何切入互联网领域，无论是切入互联网是为了分一杯羹，还是利用互联网提高公司的管理和效率，都是为了走进政府提倡的互联网+之中。

甲方：我们公司是传统国企，有大量的工人，我们想做个答题软件，答题送红包的模式，每天鼓励公司员工进行答题，学习安全管理，安全施工方面的知识。非常简单的，就有在线练习，模拟答题，正式答题，错题库，还有升级规则以及赠送红包，抽奖的几率就行。另外，题库里要分模块，因为有很多工种，答题也不一样。

乙方：这个答题软件可以做，你们具体需求呢？有整理的文档没有？

甲方：就上面我说的，还用整理文档？

乙方：还是整理一个吧，能否把升级规则和答题抽奖具体细化一下？我们还设计数据库，接口和后台逻辑。

甲方：就什么…………什么就行，就这么简单。

听的乙方一脸懵逼，这 TMD 还简单？嘴上说简单，逻辑实现起来真复杂。

乙方：好吧，那就是一个手机端，做成小程序，还需要一个后台 Web 端，你要录入题库，还要查看每个人的答题情况。价格呢，不能低于 2 万。

甲方：2 万？就这点东西，值 2万？最多 1 万 5 ，就这么定了。

乙方：1万5就1万5吧。

过了两天……

乙方，根据甲方描述整理了需求文档和开发方案，让设计师设计了一套效果图带过去了，让甲方确认一下。

甲方：没问题，挺好的，效果图方面，要是跟那个什么软件一样就好了，能否修改一下？

乙方：好的，没问题。

乙方回来，着手整理，把客户意见说给了设计师重新修改了一下，发给甲方，让甲方确认，甲方回应说：没问题。

过了几天后，乙方根据方案做的差不多了，把开发的软件给甲方试用，体验一下。

高潮来了。

甲方：哎呀，这个首页效果不好，这里少个抽奖的按钮，还有抽奖的大转盘什么的，另外，等级那里的那个宝塔设计不好，没有层次感，这个地方的颜色感觉有点冲，你们修改一下吧，我把修改意见发给你。这些问题，只有放到手机上，体验过才能发现。

乙方心里早就mmp了，艹，不是让你看过了吗？你都确认了。

乙方：好吧。

过了两天，根据客户需求，按照修改意见，修改后发给甲方，甲方，还是不满意。说：感觉还是不搭调，那个我的账户那里，把什么什么去掉，还有首页banner颜色太冲了。还有……

乙方，这里心里早就十万只草泥马奔腾而过了。忍着说：给你改。

这回，找设计师，直接跟客户沟通界面设计。

乙方设计师：你对界面都有哪些不满意的地方？有什么要求？

甲方：你先设计，我再看。

设计师心里也早就mmp了，说：你先给个意见吧，要不我不知如何修改和设计。

乙方：勉强说了一些东西。

来来回回两天又没了。

这样一个1万5的活，两周过去了，耗费了多少人力和精力？值吗？对方是传统大国企，这边是个小外包公司，不想得罪他们，因为想着可能通过这个小项目可以从那里拿个大项目的，只能忍着反复改，直到他们满意为止。

后来呢？你们是不是想知道，后台这个国企给这个小外包公司大项目了吗？

由于乙方勤勤恳恳，反复给他们修改，差点赔了倒闭掉。可是时来运转，从另外一个公司项目接到了大活。

后来，以前的那个甲方，由于从这里获得了很好的满意结果，确实送来了一个大项目。

乙方：我不接，没这个实力。

甲方：我感觉你们有，你们服务也非常好。

乙方心里早就想了，一个三天的小活给我折腾了两周，你这个半年的大项目，不得让我干三年啊，你们国企打款慢，早就被你们拖死了。

甲方：接吧，这是我们转型改革很重要的一个项目。

再后来，甲方公司由于转型互联网+不利，被国资委改革重组掉了，世上再无那个国企了。

对，这就是小外包公司撬动大公司的秘密，一般人我不告诉哦。后来，这个小公司到处说，我干掉了一个国企。看看传统企业及时转型有多么重要？如果你们有相关要求的，需要转型互联网的，有项目请联系我：非著名程序员。哈哈……

通过上面的对话，不知道大家看明白了吗？为什么现在连外包公司对于一些项目都爱答不理呢？其实原因无非以下几点：

一、沟通成本太高

由于传统企业他们不懂互联网，沟通成本这个问题，对于他们来说是看不见的，就不应该算作整个项目成本中的。这个成本我们姑且叫做隐形成本。在他们来讲，一个项目就看你做了哪些东西？而不是你和他们沟通了什么？但是对于外包公司或者承接活的一方，这个前期的沟通成本，至少要耗费一个产品经理去聊需求，而且在项目开发过程中，开发完成后，因为这个问题，那个问题，还在不断修改。本来一个周的项目能做到一个月。这就涉及到了时间成本。一个项目成本就应该是沟通成本+时间成本+开发成本。

二、其实外包项目本身很多都是低利润的

很多外包项目其实很多，大部分都是低利润的，价格要不上去，要高了，很多客户根本就不会花这个钱。但是本身我们程序员的工资又高，如果养活团队，一直靠接这种低利润的外包项目，又浪费时间，导致很多养不起开发团队。

三、好的外包公司不缺项目

其实对于好的外包公司并不缺项目，因为现在转型的企业，要做互联网项目的公司太多了，也有非常大的项目，有利润的项目在做，其实对于外包公司来说，每年接几个大项目的利润就很客观，没必要浪费在浪费时间和精力的小项目中。

一个大项目顶10个小项目，一起做大项目其实比连续做十个小项目轻松多了。尤其是和一项大项目做完之后，能够建立长期合作关系，公司利润就很可观，就可以活的很滋润。

这就是为什么连外包公司都对你爱答不理！
