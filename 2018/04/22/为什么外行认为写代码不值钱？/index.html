为什么外行认为写代码不值钱？

发表于 2018-04-22   |   分类于 程序人生   |   阅读次数

今天在 V2EX 上看到这么一个帖子，有一种感同身受的苦说不出来，所以想设身处地的聊一聊这么一个问题，尤其是我们作为开发者，程序员在接私活的时候经常会遇到这种情况，我们又该如何应对这种情况呢？

其实我们经常会遇到这样一种情况：

有朋友来找你，或者你朋友给你介绍了一个活，说要开发一个 App ，「很简单」，就是一个购物的，跟淘宝或者京东差不多？你感觉需要多少钱？

还有别的朋友可能说，我需要做一个团购功能的 App ，要求也不多，甚至「更简单」，跟美团差不多就行。你说吧，多少钱，给我开发一个。

作为行业内的人，当然知道开发一款 App ，尤其是这种集支付，商城，团购为一体的平台需要花多长时间，需要设计，前端，后台，甚至 Web 管理，这一系列的操作又得需要多少人，多少时间？可是客户不懂。

所以，为什么外行认为写代码不值钱呢？原因大概以下几点：

一、外行就是外行

既然说是外行了，那么外行就是外行，俗话说：外行看热闹，内行看门道。外行人对于编程这种专业性的东西，没有太多认识，就是感觉写代码就是写文章一样，可以很快就得完成。根本不知道什么是 UI ，什么是前端开发，后台开发，数据库设计等等。尤其是一些外行人电影可能看多了，以为一个「回车键」就跟一个回旋踢一样那么简单，暴力，快速，真实的编程哪里有电影中那么酷炫？

说白了，就是不懂。

二、市场中的模板化降低了价格门槛

另外就是一个很普通的情况，就是市面上很多软件开发公司会开会众多的模板在网上售卖，所谓的就是薄利多销的模式。一套模板的开发费用可能是几万甚至十几万，挡不住以低价 3000 一套的对外卖，导致很多人以为开发就是这么简单，你们都有现成的东西，为什么给我开发一个要这么多钱呢？

模板化的东西降低了价格的门槛，尤其是外包行业，你在接外包的网站上去看看就知道了，一个不简单的 App 开发，只需要 6000 - 8000 ，甚至更便宜，你作为一个独立开发者或者接私活的个人，是很难跟他们竞争的，因为他们模板，而你没有。你得从头开发。

当然很多外包公司出售低价竞标，他们也是有套路的，就是套路客户找他们，然后会找各种方式加价，很多客户不懂就会上当，一旦让他们开发了，总感觉投进去的钱不能不要吧，也只能接着投钱让他们开发了。

你们说，在这样的市场情况下，你能够要到高的价格吗？显然很难。

另外，我们要清楚一点，很多朋友介绍人，找独立开发者或者私人干而不找公司团队开发的目的很简单，就是为了省钱，而这样的客户显然不会提供很高的价格。所以，也不要对他们太期待，以为能够赚到很多钱。

那遇到这样的问题怎么办？

对于，遇到我刚才举的那两个例子中的客户和项目，我一般就会直接一口拒绝：做不了。你找外包公司去做吧（其实外包公司更坑人），其实作为个人来讲，要价可能比外包公司高点，但是我们诚信，不会再加价，独立开发者，程序员一般都很实在，但是外包公司里的那些业务员可不简单。

我为什么不建议程序员接私活报低价？

其实，我很不建议程序员在接私活的时候，报低价的，为什么？原因也很简单，那就是：其实客户根本就不知道要什么？很多客户在给你说开发一个东西的时候，其实自己整体需求和概念都没有，只会说一句：做的跟那个什么软件一样就行，就他们那个样子。

这时，你可能会去看那个软件的功能，你大致看来之后可能会觉得没什么，简单，但是你一旦仿着做出来之后，问题就来了。客户就会说：这不对，应该这样，那不对，应该这样。

作为产品或者程序员一定要记住这句话，那就是：用户只有在看到实物之后，才会侃侃而谈，发表各种意见。前期没有东西去谈需求，都是空谈，用户脑子里没有感念，只会附和你，说：这样就行。

最后，你报价格低的话，你会很吃亏的。客户难缠，出价越低的人，越在乎产品的好坏，反而出价越高的人，要求越低。一个连钱都斤斤计较的人，花他的钱就像扎他的心一样，你说他能不在乎产品的好坏？这是用他的血换来的啊！他都跟你斤斤计较钱了，你为什么不跟他计较呢？
