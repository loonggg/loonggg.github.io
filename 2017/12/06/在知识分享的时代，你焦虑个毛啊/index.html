在知识分享的时代，你焦虑个毛啊

发表于 2017-12-06   |   分类于 程序人生   |   阅读次数

最近浏览知乎，经常有人邀请我回答问题，类似都是一些这样的：我现在感觉好迷茫，很焦虑，总感觉自己被社会快淘汰了，经常浏览一些技术社区，订阅公众号，看博客，看到很多技术感觉自己都不会。你们说我该怎么办啊？是的，在碎片化的学习时代，人的焦虑比以前更重了。你是这样的吗？

不管你们有没有这种情况，会不会焦虑，反正作为我个人来讲，我看到很多东西如果不会的话，我会不自觉的陷入焦虑，那该如何改变这种状态呢？有一个方法非常有效，那就是：解决不了问题，就解决掉提出问题的那个人？所以，解决不了焦虑，就解决掉让你焦虑的方式。什么意思？不要杂七杂八的乱看。

在这两年进入知识分享红利时代以后，知识分享火了，很多人开始了知识分享，因此也有很多人开始订阅专栏，订阅公众号等，看大家的分享，很多分享感觉别人说的很有道理，为什么我没有想到？很多分享看了感觉自己不会的东西有很多，甚至感觉越来越多，让自己更清醒的认识了自己的无知。所以，越读越焦虑，越焦虑越迷茫，我该去向何方？

原因

造成这种情况的原因，不知道大家想过没有？是不是光想着焦虑了，没有思考一下，是什么造成了你的焦虑，仅仅是因为自己的无知吗？不见得。我认为原因大概如下：

太过于碎片化

怎么说呢？随着知识分享热潮的到来，越来越多的人开始了知识分享，再加上自媒体时代，移动互联网的迅速传播，很多人看到了这些知识，开始了迷茫，甚至手无足措的订阅，总想着感觉这些分享跟自己的技术，专业和工作相关，订阅后，能够帮助到我。

举个例子，比如：程序员，一个从事 Android 开发的程序员，仅仅目前只是从事应用层面的开发，他还经常看底层的一些技术博客和专栏，公众号等，还订阅了 Java 后台开发的公众号，博客，甚至订阅了一些 Python ，Go，人工智能方面的专栏，你说你看了这些，能不焦虑吗？你学的过来吗？估计你闲暇时间看得过来，但是学不过来，这就会造成焦虑。

这些知识，东一下，西一下，跟自己的工作看似相连，能有帮助，但是又实际可能关系不大，这么多知识，让你感觉学不过来，看到分享的人好厉害，就会感觉：人家咋这么厉害呢？

欲望过度

人啊，得学会知足。不要看着碗里的，想着锅里的，你能把自己目前负责的一亩三分地耕种好就不错了。说白了就是欲望过度，总想着一步到位，快速挣钱，感觉懂得多了值钱。我认为真正的值钱是深度，而不是广度。当然你又有深度，又有广度那当然最好了。前提是先把深度做好，再去想广度的问题。目前造成你焦虑的原因是深度没做好，不专一的同时，还总想着广度。最后深度没有，广度都是吹牛。所以，一些杂七杂八的造成你焦虑的专栏和订阅就取消吧，只订阅跟你技术相关，工作相关，有利于你深度的一些博客和专栏，公众号等。

只心动不行动

很多焦虑的人，基本上都有一个共同点，那就是：只会焦虑，不会行动。你看到了知识，不学习，没有到手，能不焦虑？道理是显而易见的，我不多说。焦虑又懒惰。你有时间悲春伤秋，真的不如着手干点什么。

其实以上所有的造成焦虑的原因总结成一句话就是：看的越多，越知道自己的无知，还不行动。我国国民在碎片化知识分享的时代，最大的矛盾不是十九大报告中说的，中国特色社会主义进入新时代，我国社会主要矛盾已经转化为人民日益增长的美好生活需要和不平衡不充分的发展之间的矛盾，而是在进入信息过载的时代，我国的主要矛盾是自己的行动跟不上自己的欲望，自己的能力跟不上技术的发展，知识的爆炸之间的矛盾。

如何看待现在的碎片化阅读

首先，我不认为碎片化阅读是学习。我认为目前碎片化的阅读顶多算是闲暇后的消遣和娱乐。这是我们需要正确看待的一点。碎片化阅读也不是没用的，我上面说了，可以给你增加一些广度，你读的东西越多，知道的可能就越多，可以在和同事，朋友之间的聊天中显示自己的知识面广，总结成两个字是：吹牛或者显摆。对，碎片化阅读可以让你吹牛和显摆。

其次，你仅仅是通过碎片化阅读可以了解和知道一些知识点，仅仅是知道，而不是自己已经掌握，学习其实是一个系统性的工程。碎片化的学习，东一点，西一点的看，不能说没用，但是由于大众没有串接起来的能力，导致你不能完全掌握和驾驭它。所以在用的时候，可能还是会手足无措，这样只会增加自己的焦虑。所以啊，我建议大家学习东西的时候，最好看书。书是一个系列和系统的工程，而看博客和专栏，公众号都是零零散散的点，点多了可能是成面了，但是凌乱的点组合差了就是错误的系统“面”，这是非常危险的。看书学习是主动型的学习，而看视频，订阅等推荐式的学习是被动型的学习，被动不如主动，多点主动型的比较好。

最后，我建议大家要正确看待这个碎片化的阅读和订阅，学习是系统性的工程，目前我们很多人都开始做技术公众号，技术分享，但是大家在阅读和看的时候，不要产生一些焦虑，也不要感觉分享的人很厉害，不一定，说不定没你厉害，术业有专攻，大家各有优点和能力。程序员分很多种，每种又有很多方向，术业有专攻，仅此而已，都有自己擅长不擅长的领域。建议大家去订阅跟自己工作比较密切，有利自己深度拓展的专栏，博客等，少看一些提高广度的。先深后广，相信自己深度好了就完全没问题了，广度算是附加价值，深度才是核心价值。

看完这些分析，是不是心里好受了一些？所以啊，焦虑个毛啊，别焦虑了，别有什么知识恐慌，明明是知识过载，我们按着一个方向攻才是容易成功的，而非全才，全栈。
